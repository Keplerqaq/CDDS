
# 开题报告：小猫钓鱼游戏 + 世界各国制造业增加值统计分析系统

---

## 一、选题的意义

本课题来源于数据结构课程设计第 3 组，包含两个子题目，分别对应数据结构基础阶段和高级阶段的核心知识。

**题目一：小猫钓鱼游戏**

小猫钓鱼是一种经典的扑克牌博弈游戏。其出牌入队尾、收牌入队尾的手牌操作是典型的队列模型（FIFO）；牌叠入栈、从顶部收走的桌面操作是典型的栈模型（LIFO）。选题的核心价值在于——用链式存储实现这两个抽象数据类型，将课堂上学到的队列和栈从"知道概念"推进到"能独立设计和串联运行"。

**题目二：制造业增加值统计分析系统**

制造业是国民经济的支柱产业。利用 1999-2019 年 96 个国家的真实面板数据，完成数据导入、增速计算、排序、统计分析等任务，既是对排序算法和顺序表操作的综合应用，也能通过排名直观感受中国制造业在全球格局中的变迁，让数据结构的训练不脱离现实意义。

**综合意义**：两个题目覆盖了本课程已掌握的绝大部分数据结构——顺序表、链表、栈、队列、排序、查找，是一次从"会写单个算法"到"能搭建完整系统"的综合训练。

## 二、问题分析

### 2.1 小猫钓鱼游戏

**输入**：一副扑克牌（1-9 共 N 张，N 在洗牌前确定），随机洗牌后平分给甲、乙二人。

**过程**：
- 甲先出一张牌，放到桌面上
- 乙出一张牌，叠到桌面上
- 交替进行。每一轮出牌后判断：当前出的这张牌在桌面上是否已存在？
  - **存在**：自己出的这张牌直接入队尾 → 从桌面顶部（栈顶）连续出栈并依次入队尾 → 直到遇到与出牌相同的那张旧牌 → 旧牌也出栈入队尾 → 本轮结束
  - **不存在**：压栈，标记数组中该牌面标记为 1

**输出**：一方手牌为空时游戏结束，宣布胜者。

**关键不变量**：桌上每种牌面至多一张。因为出现重复时立刻收走，所以 `table_flag[10]` 布尔数组就够，不需要计数。

**边界情况**：
- 牌面范围固定为 1-9
- 可能无胜负（死循环）——需设置回合上限防止无限对局
- 初始手牌是随机分配，每局不同

### 2.2 制造业增加值统计分析系统

**输入**：文本文件，96 国 × 21 年（1999-2019）制造业增加值数据，附收入等级标签。

**七项功能需求**：

| 功能 | 输入 | 输出 | 关键约束 |
|------|------|------|----------|
| ① 数据导入 | 文本文件 | 内存中的顺序表 | 96 国保证不超 MAXSIZE |
| ② 查询 | 国家名 + 年份 | 增加值 + 增速 | 国家名全字匹配，年份范围 1999-2019 |
| ③ 增速计算 | 原始增加值 | 覆盖 growth_rate 数组 | 上年为 0 则增速记为 0 |
| ④ 增加值排名 | 增加值 | index_va 索引数组 | 不能移动原始记录，用快速排序 |
| ⑤ 增速排名 | 增速 + 收入等级 | index_gr 索引数组 | 先分组再组内排，用选择排序，与④不同算法 |
| ⑥ 增加值分析 | 国家名 | min/max/均值/方差 | 方差用样本方差（n-1） |
| ⑦ 保存结果 | 排序结果 | 两个 txt 文件 | 文件名由原始文件名派生 |

**核心技术难点**：索引数组。教材明确要求"排序不能改变原始记录的物理顺序，也不能新建一个排好序的记录序列"，因此排名必须通过索引数组间接实现——排的是索引值，比较时通过索引间接访问原始数据，记录不动。

## 三、存储结构及核心算法接口设计

### 3.1 小猫钓鱼游戏

**存储结构**：

```
链式队列（LinkQueue）
├── front → QNode(card=5) → QNode(card=2) → QNode(card=7) → NULL
└── rear ──────────────────────────────────────────────────┘

链式栈（LinkStack）
└── top → SNode(card=3) → SNode(card=6) → SNode(card=1) → NULL

标记数组
└── table_flag[10] = {0, 1, 0, 1, ...}  // table_flag[i]==1 表示桌面有 i
```

**核心接口**：

```c
// 队列
void queue_init(LinkQueue *q);           // 初始化空队列
int  queue_is_empty(LinkQueue *q);       // 判空
void enqueue(LinkQueue *q, int card);    // 入队（尾部）
int  dequeue(LinkQueue *q);              // 出队（头部）

// 栈
void stack_init(LinkStack *s);           // 初始化空栈
int  stack_is_empty(LinkStack *s);       // 判空
void push(LinkStack *s, int card);       // 压栈
int  pop(LinkStack *s);                  // 出栈

// 辅助
void shuffle(int arr[], int n);          // Fisher-Yates 洗牌
void deal(LinkQueue *a, LinkQueue *b, int arr[], int n); // 发牌
```

**选择链式而非数组的理由**：手牌张数在游戏过程中动态增减（收牌时大量牌入队），链式结构不需要预分配固定容量，也不会像数组队列那样出现"假溢出"。

### 3.2 制造业增加值统计分析系统

**存储结构**：

```
顺序表 SqList
├── r[1..96]: RecType 记录数组（r[0] 不用）
│     ├── country[30], country_type
│     ├── value_added[21], growth_rate[21], year[21]
│     ├── index_va[21]   // 增加值排名（逐年）
│     └── index_gr[21]   // 增速排名（逐年，组内）
├── index_l[]:   低收入国家在 r[] 中的下标
├── index_ml[]:  中低收入国家在 r[] 中的下标
├── index_mh[]:  中高收入国家在 r[] 中的下标
├── index_h[]:   高收入国家在 r[] 中的下标
└── length: 96
```

**为什么用顺序表**：数据量固定（96 国 × 21 年），不需要动态扩容；顺序存储天然支持按国家名查找和按索引访问，和索引数组的间接寻址配合最自然。

**核心接口**：

```c
// ① 数据导入
void MVA_SqList_Read(SqList *L, const char *filename);
//   从 txt 逐行读取，解析国家名、收入等级、21 年数据，填充顺序表

// ② 查询
void MVA_SqList_Search(SqList *L);
//   输入国家名 + 年份 → 遍历顺序表匹配 → 输出增加值 + 增速

// ③ 增速计算
void MVA_SqList_Calculate(SqList *L);
//   growth_rate[k] = (val[k] - val[k-1]) / val[k-1]，k>=1
//   val[k-1]==0 时 growth_rate[k]=0

// ④ 增加值排名（快速排序）
void MVA_SqList_Sort_Va(SqList *L);
//   逐年快排索引数组，降序，结果写入 index_va
int  partition(SqList *L, int a[], int low, int high, int year);
void quick_sort_idx(SqList *L, int a[], int low, int high, int year);

// ⑤ 增速排名（选择排序，分组）
void MVA_SqList_Sort_Gr(SqList *L);
//   先按收入等级分四组 → 各组逐年选择排序降序 → 结果写入 index_gr
void group_sort_select(SqList *L, int *group, int size, int *result, int year);

// ⑥ 增加值分析
void MVA_SqList_Analyze(SqList *L);
//   输入国家名 → 计算 min/max/均值/样本方差 → 输出

// ⑦ 保存
void MVA_SqList_Save(SqList *L, const char *src_name);
//   导出增加值排名到 _Sorted.txt，增速排名到 _Grouped_Sorted.txt
```

**两种排序算法对比**：

| 维度 | 增加值排名（快排） | 增速排名（选择排序） |
|------|-------------------|---------------------|
| 时间复杂度 | O(n log n) 平均，O(n²) 最坏 | O(n²) 始终 |
| 空间复杂度 | O(log n) 递归栈 | O(1) |
| 稳定性 | 不稳定 | 不稳定 |
| 适用场景 | 全局 96 国全量排序 | 四组子序列（每组约 10-50 国） |
| 选型依据 | 数据量适中，效率优先 | 分组后规模小，实现简洁 |
