# 小猫钓鱼游戏 —— 逐行代码讲解

## 一、数据结构定义

### 链式队列 QNode + LinkQueue

```c
typedef struct QNode {
    int card;
    struct QNode next;
} QNode, *PQNode;
```

`QNode` 是队列中的一个节点——存一张牌。`card` 是牌面值（1–9）。`next` 指向下一张牌。这是链表的经典节点定义。

**为什么要用链表？** 手牌张数在游戏过程中动态增减——收牌时可能一次收十几张，不能预设数组大小。链式结构按需分配，永远不会"装不下"。

```c
typedef struct {
    PQNode front;
    PQNode rear;
} LinkQueue, *PLinkQueue;
```

`LinkQueue` 是链式队列。`front` 指向队头（出牌从这里取），`rear` 指向队尾（收牌加到后面）。

**为什么需要两个指针？** 队列是两端操作——FIFO（先入先出）。如果只有 front，入队要遍历整个链表找尾巴，每次都是 O(n)。有了 rear，入队直接追加到 rear 后面，O(1)。

**后续用在哪**：两个玩家各有一个 LinkQueue（`player_a`、`player_b`），分别存储甲乙的手牌。所有出牌、收牌操作都通过队列接口完成。

---

### 链式栈 SNode

```c
typedef struct SNode {
    int card;
    struct SNode *next;
} SNode, *PSNode;
```

`SNode` 是栈节点。`card` 存牌面值，`next` 指向下面的牌。

```c
typedef struct {
    PSNode top;
} LinkStack, *PLinkStack;
```

**为什么栈也加了 LinkStack 包装？** 和队列一样，用包装结构消除二级指针。`push(PLinkStack s, int card)` 和 `enqueue(PLinkQueue q, int card)` 写法完全统一——都是一级指针，看一眼就懂。`top` 指向栈顶节点，空栈时 `top = NULL`。

## 二、队列操作

### queue_init —— 初始化空队列

```c
void queue_init(PLinkQueue q) {
    q->front = q->rear = NULL;
}
```

front 和 rear 都设为 NULL，表示空队列。**后续所有判空操作都检查 `q->front == NULL`**。

---

### queue_is_empty —— 判空

```c
int queue_is_empty(PLinkQueue q) {
    return q->front == NULL;
}
```

空队列的特征：没有节点，front 为 NULL。注意 rear 此时也是 NULL，但判空只看 front——如果只看 rear，在有元素时 rear 指向最后一个节点，不满足"空"的判断逻辑。

**后续用在哪**：`play_turn` 里不需要（出牌前不会检查——如果手牌空了说明游戏已结束）。`queue_destroy` 用它判断是否全释放完。游戏结束后用它判断玩家是否还有剩余手牌。

---

### enqueue —— 入队（加到队尾）

```c
void enqueue(PLinkQueue q, int card) {
    PQNode node = (PQNode)malloc(sizeof(QNode));
    node->card = card;
    node->next = NULL;
```

`malloc` 分配一个新节点。**为什么用 malloc 而不是栈变量？** 栈变量在函数返回后就失效，malloc 在堆上分配，节点生命周期由程序控制——直到调用 `free`（在 `dequeue` 中释放）才消失。每张牌"存活"多久是不确定的，所以必须堆分配。

```c
    if (queue_is_empty(q)) {
        q->front = q->rear = node;
    } else {
        q->rear->next = node;
        q->rear = node;
    }
}
```

**空队列**：新节点既是队头也是队尾（队列里只有它一个）。  
**非空队列**：挂在当前队尾后面，然后 rear 移到新节点。

**链式队列入队的核心操作**：只动 rear，不动 front。

**后续用在哪**：发牌时 36 张牌平分入队；收牌时赢到的牌全部从队尾入队。

---

### dequeue —— 出队（从队头取出）

```c
int dequeue(PLinkQueue q) {
    PQNode tmp = q->front;
    int card = tmp->card;
    q->front = q->front->next;
    if (q->front == NULL) q->rear = NULL;
    free(tmp);
    return card;
}
```

1. 记住队头节点 `tmp`。
2. 取出牌面值 `card`（返回值）。
3. `front` 指向下一张牌——队头出队。
4. **关键**：如果出队后队列空了（`front == NULL`），rear 也要置 NULL。否则 rear 指向一个已释放的内存块，下次入队会出现悬空指针错误。
5. `free(tmp)` 释放节点。不释放就会内存泄漏——每出一张牌泄漏一个 `sizeof(QNode)` 字节。

**后续用在哪**：`play_turn` 中玩家每次出牌调用 `dequeue`。

---

### queue_length —— 求队列长度

```c
int queue_length(PLinkQueue q) {
    int len = 0;
    PQNode p = q->front;
    while (p) { len++; p = p->next; }
    return len;
}
```

从头遍历到尾，数节点个数。**为什么不能像数组队列一样 O(1) 取值？** 链式结构没有 `length` 字段。如果要频繁查询长度，可以在 LinkQueue 里加一个 `int length` 并用 enqueue/dequeue 维护。但这游戏里只展示手牌状态，不是性能瓶颈，遍历就够。

**后续用在哪**：每轮结束后打印当前手牌张数。

---

### queue_print —— 打印队列

```c
void queue_print(PLinkQueue q) {
    PQNode p = q->front;
    if (!p) { printf("（空）"); return; }
    while (p) {
        printf("%d", p->card);
        p = p->next;
        if (p) printf(", ");
    }
}
```

从头遍历打印。`if (p)` 是判断"后面还有没有牌"，决定要不要输出逗号分隔。

**后续用在哪**：每轮出牌后显示当前手牌内容。

---

### queue_destroy —— 销毁队列

```c
void queue_destroy(PLinkQueue q) {
    while (!queue_is_empty(q)) dequeue(q);
}
```

循环出队直到空。`dequeue` 每次会 `free` 一个节点，所以最终所有 malloc 的内存都被回收。**不调用这个函数会导致内存泄漏**。

**后续用在哪**：游戏结束后清理两个玩家的手牌。

---

## 三、栈操作

### stack_init —— 初始化空栈

```c
void stack_init(PLinkStack s) {
    s->top = NULL;
}
```

和 `queue_init` 完全一致——置空。

---

### push —— 压栈

```c
void push(PLinkStack s, int card) {
    PSNode node = (PSNode)malloc(sizeof(SNode));
    node->card = card;
    node->next = s->top;
    s->top = node;
}
```

**为什么是 `PLinkStack s` 而不是二级指针？** 包装结构 `LinkStack` 包住了 `top` 指针，`s->top` 可以直接修改结构体内部的指针字段，不需要二级指针。

**工作原理**：新节点插入链表头部。`node->next = s->top` 让新节点指向原来的栈顶，然后 `s->top = node` 让栈顶指向新节点。和 `enqueue` 写法统一——都传一级指针。

**后续用在哪**：出牌时桌面上没有这张牌 → 压栈（把牌放在桌面牌堆顶部）。

---

### pop —— 出栈

```c
int pop(PLinkStack s) {
    PSNode tmp = s->top;
    int card = tmp->card;
    s->top = s->top->next;
    free(tmp);
    return card;
}
```

和 `dequeue` 写法一致——记住栈顶、取出牌值、栈顶下移、释放旧节点。

**后续用在哪**：出牌时桌面已有这张牌 → 从栈顶连续 pop，把收走的牌放入队尾。

---

### stack_is_empty —— 栈判空

```c
int stack_is_empty(PLinkStack s) {
    return s->top == NULL;
}
```

和 `queue_is_empty` 一样——检查是否为空。

---

### stack_print —— 打印栈

```c
void stack_print(PLinkStack s) {
    if (!s->top) { printf("（空）"); return; }
    PSNode p = s->top;
    while (p) {
        printf("%d", p->card);
        p = p->next;
        if (p) printf(", ");
    }
}
```

从栈顶到栈底依次打印。栈顶是链表头，所以打印顺序就是"最后放的牌在前"——符合"桌面从上到下"的视觉效果。

---

### stack_destroy —— 销毁栈

```c
void stack_destroy(PLinkStack s) {
    while (!stack_is_empty(s)) pop(s);
}
```

和 `queue_destroy` 写法一致——循环出栈直到空。

---

## 四、辅助函数

### shuffle —— 洗牌（Fisher-Yates 算法）

```c
void shuffle(int deck[], int n) {
    for (int i = n - 1; i > 0; i--) {
        int j = rand() % (i + 1);
        int tmp = deck[i];
        deck[i] = deck[j];
        deck[j] = tmp;
    }
}
```

**为什么选 Fisher-Yates？** 这是"公平洗牌"的标准算法——每种排列出现的概率均等。

**原理**：从后往前，对于每个位置 i，在 `[0, i]` 范围内随机选一个位置 j，交换 `deck[i]` 和 `deck[j]`。i 从 n-1 递减到 1（只剩一个元素时不需要交换）。

**`rand() % (i + 1)` 中的 `%`**：取余运算，把随机数映射到 `[0, i]` 范围内。

**后续用在哪**：游戏开始前将 36 张牌打乱。

---

### deal_cards —— 发牌

```c
void deal_cards(int deck[], int deck_size,
                PLinkQueue player_a, PLinkQueue player_b) {
    int half = deck_size / 2;
    for (int i = 0; i < half; i++)
        enqueue(player_a, deck[i]);
    for (int i = half; i < deck_size; i++)
        enqueue(player_b, deck[i]);
}
```

将洗好的牌平分：前一半归甲，后一半归乙。`deck_size / 2` 整除（36/2=18）。每人 18 张，通过 enqueue 依次入队。

---

### play_turn —— 执行一轮出牌

```c
int play_turn(PLinkQueue player, PLinkStack table,
              int *flag, const char *who) {
```

参数：`player` 是当前出牌者的手牌队列，`table` 是桌面牌堆的栈顶指针，`flag` 是标记数组（查重用），`who` 是字符串 "甲" 或 "乙"（用于打印）。

**返回值**：返回 `queue_is_empty(player)` 的结果——如果出完牌后手牌为空，返回 1（游戏结束）。

```c
    int card = dequeue(player);
    printf("  %s出 [%d] → ", who, card);
```

出队列头部的一张牌。**为什么是 dequeue 而不是从队列中间拿？** 游戏规则：每次出手中的第一张。队列的头部就是"第一张"。

```c
    if (flag[card] == 1) {
```

**查重核心**：`flag[card]` 是布尔标记——card 这张牌桌面上有没有？O(1) 查重！不需要遍历整个栈去找同牌面的牌。

**为什么可以这样？** 游戏有个不变量：桌上每种牌面至多一张。因为一旦出现相同，立刻收走。所以 flag 用布尔值就够了，不需要计数。这就是用空间换时间——10 个 int 换 O(1) 查找。

```c
        printf("桌上已有 %d，收牌！", card);
        enqueue(player, card);
```

**关键**：自己出的这张牌也收回去！先入队尾。然后开始收桌上的牌。

```c
        int collected = 1;
        while (1) {
            int top = pop(table);
            flag[top] = 0;
            enqueue(player, top);
            collected++;
            if (top == card) break;
        }
        printf(" 共收 %d 张", collected);
```

**收牌逻辑**：
1. 从栈顶 pop 一张牌。
2. `flag[top] = 0` —— 这张牌从桌面移走了，标记清掉。
3. 入队尾（enqueue）。
4. 如果 pop 出来的牌就是自己刚出的那张（`top == card`），停止。这意味着"两张相同牌之间的全部牌 + 这两张牌自己"都被收走。

**注意 `flag` 的同步维护**：每次 pop 出栈立即清零对应标记。这样栈和标记数组始终保持一致。不维护的话，下次查重就会误判。

**collected** 从 1 开始是因为自己刚出的牌其实也算收走了（先 enqueue 了）。count 只用于打印"收了几张"。

```c
    } else {
        printf("无匹配，留在桌面");
        push(table, card);
        flag[card] = 1;
    }
```

无相同牌：压到栈顶，标记置 1。

```c
    printf("  |  手牌[%d]: ", queue_length(player));
    queue_print(player);
    printf("\n");
    return queue_is_empty(player);
}
```

打印当前手牌状态，返回"手牌是否为空"。**返回值是整个游戏循环的终止判断**——如果某方手牌为空，对手获胜。

### fish_game —— 主游戏循环

```c
void fish_game(void) {
    srand((unsigned int)time(NULL));
```

`time(NULL)` 返回当前时间戳（秒），转为 unsigned int 作为随机种子。**为什么需要这个？** `rand()` 是伪随机数生成器，每次都从固定种子开始的话，每次洗牌结果一模一样。用时间做种子保证每局牌面不同。

```c
    int deck[36], deck_size = 36, idx = 0;
    for (int v = 1; v <= 9; v++)
        for (int i = 0; i < 4; i++)
            deck[idx++] = v;
```

**构造牌堆**：牌面 1-9，每种 4 张（相当于 4 种花色但不区分），共 36 张。嵌套循环：外层 9 种牌面，内层每种生成 4 张。

**为什么是 1-9 而不是 1-13？** 教材规定"为了简化实现，约定牌面值只有 1-9"。每种 4 张总计 36 张刚好平分。

```c
    shuffle(deck, deck_size);
```

洗牌——把 deck[] 随机打乱。

```c
    LinkQueue player_a, player_b;
    queue_init(&player_a);
    queue_init(&player_b);
    deal_cards(deck, deck_size, &player_a, &player_b);
```

创建两个玩家的手牌队列，初始化后发牌。

```c
    LinkStack table;
    stack_init(&table);
    int table_flag[10] = {0};
```

`table = NULL`：桌面牌堆初始为空栈。  
`table_flag[10] = {0}`：标记数组全初始化为 0（表示桌面没有牌）。大小为 10（索引 0-9），牌面 1-9 正好用 1-9 下标。**下标 0 闲置不用**，这样 `flag[card]` 中 card 就是牌面值，不用 `card-1` 转换，代码直观。

```c
    printf("初始手牌：\n");
    printf("  甲 (%d张): ", queue_length(&player_a));
    queue_print(&player_a);
    printf("\n  乙 (%d张): ", queue_length(&player_b));
    queue_print(&player_b);
    printf("\n\n");
```

展示初始状态——双方各有多少张、分别是什么。

```c
    int round = 0;
    while (1) {
        round++;
        printf("第 %d 轮：", round);
```

无限循环直到某一方手牌出完。`round` 计数器用于显示"第几轮"。虽然没有强制设回合上限，但实际 36 张牌必然会在有限回合内结束。

```c
        if (play_turn(&player_a, &table, table_flag, "甲")) {
            printf("\n甲手牌出完，乙获胜！\n"); break;
        }
```

甲先出牌（游戏规定）。如果 `play_turn` 返回 1（甲手牌已空）→ 乙获胜，break 退出循环。

```c
        printf("       ");
        if (play_turn(&player_b, &table, table_flag, "乙")) {
            printf("\n乙手牌出完，甲获胜！\n"); break;
        }
        printf("\n");
    }
```

空格缩进让甲和乙的输出在视觉上错开。乙出牌，如果乙手牌空 → 甲获胜。

**为什么甲先手？** 规则——"玩家甲先拿出手中的第一张扑克牌放在桌上"。甲有先手优势。

```c
    printf("\n========================================\n");
    printf("              游戏结束\n");
    printf("========================================\n");
    printf("桌面剩余牌（栈顶→栈底）: ");
    stack_print(&table);
    printf("\n");
    if (!queue_is_empty(&player_a)) {
        printf("甲方手牌: "); queue_print(&player_a); printf("\n");
    }
    if (!queue_is_empty(&player_b)) {
        printf("乙方手牌: "); queue_print(&player_b); printf("\n");
    }
```

打印最终状态：桌面剩余牌（若有）、双方剩余手牌（胜利方可能有牌，失败方一定为空）。

```c
    queue_destroy(&player_a);
    queue_destroy(&player_b);
    stack_destroy(&table);
}
```

**释放全部动态内存**。malloc 出去的每个 QNode 和 SNode 都在 dequeue/pop 中被 free。不调用这三行会导致内存泄漏——虽然程序退出后操作系统会回收，但在正式项目中是坏习惯。养成"malloc 必对应 free"的肌肉记忆。

## 五、数据结构与游戏规则的映射总览

| 游戏场景 | 实际操作 | 数据结构 | 操作接口 |
|---------|---------|---------|---------|
| 手牌 | 出牌从前面出，收牌加到后面 | 链式队列 LinkQueue | enqueue / dequeue |
| 桌面 | 放牌叠上去，收牌从顶上拿 | 链式栈 SNode* | push / pop |
| 查重 | 某张牌桌面上有没有？ | 标记数组 table_flag[10] | flag[card] == 1 |
| 洗牌 | 36 张牌随机打乱 | 数组 deck[36] | Fisher-Yates shuffle |
| 发牌 | 平分给甲乙二人 | 队列 enqueue | deal_cards |

**为什么链式而不用数组？**

数组队列有"假溢出"问题——出队后 front 前的空间无法复用（除非做成循环队列）。链式队列动态分配，出队即释放，无论游戏持续多少轮都不会浪费内存。

数组栈同样需要预设最大容量。但游戏最长能玩多少轮不确定——理论上双方可能来回收牌很久。链式栈按需增减，没有上限。

## 六、执行流程

```
1. 构造 36 张牌（1-9 各 4 张）
2. srand(time) 设置随机种子
3. Fisher-Yates 洗牌
4. 平分发牌：甲 18 张（队列），乙 18 张（队列）
5. 打印初始手牌

6. while(1):
   甲出牌 → dequeue
     ├─ flag[card]==1 → 收牌！（自己牌入队，栈顶连续 pop 入队到匹配为止）
     └─ flag[card]==0 → 压栈，flag[card]=1
    出完后手牌空了？→ 乙获胜，break

   乙出牌 → dequeue
     ├─ flag[card]==1 → 收牌！
     └─ flag[card]==0 → 压栈，flag[card]=1
    出完后手牌空了？→ 甲获胜，break

7. 打印最终状态
8. 销毁队列和栈（释放内存）
```
