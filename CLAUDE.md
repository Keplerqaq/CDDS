### 题目一：小猫钓鱼游戏

核心数据结构：
- 链式队列（玩家手牌）— 出牌出队头、收牌入队尾，FIFO
- 链式栈（桌面牌堆）— 放牌压栈、收牌出栈，LIFO
- 标记数组 `int table_flag[10]` — O(1) 查重，布尔值即可（游戏不变量：桌上任意时刻每种牌面最多一张，出现相同立刻收走）

流程：甲先出一张 → 乙出一张叠上 → 交替。出牌时查 `table_flag[card] == 1`：
- **有相同牌**：自己出的牌直接入队尾 → 从栈顶连续出栈，`table_flag` 同步清零 → 遇到匹配的旧牌后停止 → 这些牌全部入队尾
- **无相同牌**：压栈，`table_flag[card] = 1`

收牌不压栈！牌面 1-9，总张数需在洗牌前明确。

### 题目二：制造业增加值统计分析系统

数据来源：`制造业分析_inputdate.txt`，96 个国家，每年一行含国家名、收入等级（低收入/中低等收入/中高等收入/高收入）、1999-2019 共 21 年增加值，2010 年不变价美元折算，缺失填 0。

核心数据结构：顺序表 + 索引数组。教材硬性约束——**排序不能改变原始记录顺序，也不能新建记录序列**，因此排名信息必须通过索引数组存储位置映射。

```
struct Record {
    char country[30];
    int  country_type;          // 0=低收入 1=中低等收入 2=中高等收入 3=高收入
    float value_added[21];      // 1999-2019 各年制造业增加值（亿美元）
    float growth_rate[21];      // 同比增速
    int   year[21];             // 年份标签 1999-2019
    int   index_va[21];         // 增加值排名索引（逐年）
    int   index_gr[21];         // 增速排名索引（逐年）
};
```

七项功能（教材第 6 章"内排序"，页码 241）：

① **数据导入**：从 txt 文件批量导入 96 国 21 年数据。  
② **查询**：按用户输入的国家名 + 年份查找，输出对应增加值和增速。  
③ **增速计算**：公式为 `(当年-上年)/上年`，上年为 0 则增速记为 0。增速覆盖 2000-2019。  
④ **增加值排名**：逐年对所有国家按增加值降序排。结果存入 `index_va`。  
⑤ **增速排名**：先按收入等级分四组子序列，再对各组逐年按增速降序排。结果存入 `index_gr`。④⑤必须使用**不同的排序算法**。  
⑥ **增加值分析**：对指定国家计算增加值和增速的 min/max/均值/方差。方差公式为样本方差 `S² = Σ(Xi-X̄)²/(n-1)`。  
⑦ **保存**：增加值排名存为 `文件名_Sorted.txt`，增速排名存为 `文件名_Grouped_Sorted.txt`。

索引数组原理（不改变原始顺序）：  
初始 `index_va = [1,2,...,96]`，排序时比较 `r[index_va[i]].value_added[year]`，交换只换 index_va 的值（不移动 r[]），排完后 `r[index_va[1]]` 就是第一名。

已知参考代码（`参考代码/制造业分析.txt`）采用选择排序（增加值排名）+ 选择排序（增速排名，已按收入等级分组），索引数组实现。我们的实现需将其中一个排名改用快速排序，以满足"不同算法"要求。
